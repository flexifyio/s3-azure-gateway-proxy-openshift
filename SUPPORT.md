# Support

Thank you for using s3-azure-gateway-proxy-openshift! We're here to help.

## Getting Help

### Questions & General Support

For general questions about deployment, configuration, or usage:

- **[GitHub Discussions](https://github.com/flexifyio/s3-azure-gateway-proxy-openshift/discussions)**: Ask questions and share experiences with the community
- **[Issues](https://github.com/flexifyio/s3-azure-gateway-proxy-openshift/issues)**: Check existing issues - your question might already be answered
- **[Flexify Support](https://flexify.io/#contacts)**: Questions related to Flexify.IO

### Documentation

- **[README.md](README.md)**: Quick start, installation, and basic usage
- **[CONTRIBUTING.md](CONTRIBUTING.md)**: How to contribute and development setup
- **[Configuration Guide](CONFIGURATION.md)** (coming soon): Detailed configuration options
- **[Troubleshooting Guide](TROUBLESHOOTING.md)** (coming soon): Common issues and solutions

### Bugs & Issues

Found a bug? Help us fix it:

1. [Search existing issues](https://github.com/flexifyio/s3-azure-gateway-proxy-openshift/issues) to see if it's already reported
2. If not found, [create a new issue](https://github.com/flexifyio/s3-azure-gateway-proxy-openshift/issues/new?template=bug_report.md) with:
   - Clear description
   - Steps to reproduce
   - Expected vs actual behavior
   - Your environment (OS, K8s version, etc.)
   - Logs or error messages

### Feature Requests

Have an idea for improvement?

1. [Check existing discussions](https://github.com/flexifyio/s3-azure-gateway-proxy-openshift/discussions) and [issues](https://github.com/flexifyio/s3-azure-gateway-proxy-openshift/issues)
2. [Create a feature request](https://github.com/flexifyio/s3-azure-gateway-proxy-openshift/issues/new?template=feature_request.md) with:
   - What problem it solves
   - Your proposed solution
   - Alternative approaches considered
   - Why it would be valuable

### Security Issues

For security vulnerabilities, please do NOT create a public issue.

See [SECURITY.md](SECURITY.md) for:
- Responsible disclosure process
- Private reporting methods
- Response timeline expectations

## Self-Help Checklist

Before opening an issue, try:

- [ ] Read the README completely
- [ ] Check documentation and guides
- [ ] Search existing issues for similar problems
- [ ] Review error messages and logs carefully
- [ ] Try troubleshooting steps if available
- [ ] Check your configuration against examples
- [ ] Verify prerequisites are met
- [ ] Test in isolation if possible

## Providing Good Information

The better information you provide, the faster we can help:

### For Questions
- What you're trying to accomplish
- What you've already tried
- Error messages or logs
- Your environment details

### For Bug Reports
- Steps to reproduce consistently
- Expected vs actual behavior
- Environment (OS, version, etc.)
- Relevant logs and error messages
- Screenshots or examples if applicable

### For Feature Requests
- Why you need this feature
- Your proposed approach
- Alternative solutions considered
- Use cases and benefits

## Contributing Solutions

If you've found a solution to an issue:

- **Share in discussions**: Help other users facing the same problem
- **Contribute code**: Submit a PR with the fix (see [CONTRIBUTING.md](CONTRIBUTING.md))
- **Improve documentation**: Help document solutions and workarounds

## Escalation

If you feel your issue isn't getting adequate attention:

1. Add a comment with additional details
2. @mention maintainers if appropriate
3. Open a new discussion linking to original issue
4. See [GOVERNANCE.md](GOVERNANCE.md) for escalation procedures

- Contributing solutions

Respectful, constructive communication helps everyone!
